# Multi-Brand White-Label App - מערכת ניהול מכון לק

אפליקציה לניהול מכון לק עם מערכת התחברות מתקדמת ותמיכה בסוגי משתמשים שונים. האפליקציה תומכת במספר לקוחות (white-label) עם עיצוב ומותגים שונים.

## תכונות עיקריות

### 🔐 מערכת התחברות
- עמוד התחברות מודרני ויפה
- תמיכה בשני סוגי משתמשים:
  - **משתמש רגיל** - גישה למסכים הרגילים
  - **משתמש מיוחד** - גישה למסכים מתקדמים עם תפריט תחתון שונה

### 📱 מסכים למשתמש רגיל
- דף בית
- יומן תורים
- ניהול לקוחות
- הגדרות
- פרופיל

### ⭐ מסכים למשתמש מיוחד
- דף בית מיוחד
- לוח בקרה מתקדם
- דוחות מפורטים
- הגדרות מתקדמות
- פרופיל מיוחד

## התקנה והרצה

### דרישות מקדימות
- Node.js (גרסה 18 ומעלה)
- npm או bun
- Expo CLI

### התקנה
```bash
# התקנת תלויות
npm install

# או עם bun
bun install
```

### הרצה

#### הרצה עם לקוח ספציפי
```bash
# הרצה עם לקוח A
npm run start:clientA

# הרצה עם לקוח B
npm run start:clientB

# הרצה עם לקוח ברירת מחדל
npm start

# הרצה עם לקוח A (web)
npm run start:web:clientA

# הרצה עם לקוח B (web)
npm run start:web:clientB
```

#### הרצה רגילה
```bash
# הרצת האפליקציה
npm start

# או עם bun
bun start
```

### בניית אפליקציה ללקוח ספציפי
```bash
# בניית אפליקציה ללקוח A (iOS)
npm run build:clientA:ios

# בניית אפליקציה ללקוח A (Android)
npm run build:clientA:android

# בניית אפליקציה ללקוח A (שני הפלטפורמות)
npm run build:clientA:all

# בניית אפליקציה ללקוח B (iOS)
npm run build:clientB:ios

# בניית אפליקציה ללקוח B (Android)
npm run build:clientB:android

# בניית אפליקציה ללקוח B (שני הפלטפורמות)
npm run build:clientB:all

# בנייה מותאמת אישית
node scripts/build-client.mjs <client> <platform> <buildProfile>
# דוגמה: node scripts/build-client.mjs clientA ios production
```

### בדיקת המערכת
```bash
# בדיקת מערכת ה-theme
npm run test:theme

# בדיקת מערכת הלוגואים
npm run test:logos
```

## מערכת White-Label

האפליקציה תומכת במספר לקוחות עם עיצוב ומותגים שונים:

### 🎨 התאמה אישית לכל לקוח
- **צבעים מותאמים** - כל לקוח יכול להגדיר צבעים ייחודיים
- **לוגו מותאם** - אייקון ולוגו מותאמים לכל לקוח
- **לוגו דינמי** - לוגו מתחלף אוטומטית במסכי הבית של הלקוח והמנהל
- **שם אפליקציה** - שם מותאם לכל לקוח
- **מסך טעינה** - מסך טעינה מותאם לכל לקוח
- **מזהי אפליקציה** - bundle identifier ו-package name ייחודיים

### 📱 בניית אפליקציות נפרדות
- כל לקוח מקבל אפליקציה נפרדת עם מזהים ייחודיים
- תמיכה ב-iOS ו-Android
- בנייה אוטומטית עם EAS Build
- ניהול גרסאות נפרד לכל לקוח

### 🔧 הוספת לקוח חדש
1. צור תיקייה חדשה ב-`branding/<clientName>/`
2. הוסף קבצים:
   - `app.config.json` - הגדרות האפליקציה
   - `theme.json` - עיצוב וצבעים
   - `icon.png` - אייקון האפליקציה
   - `splash.png` - מסך טעינה
3. הוסף לוגו ל-`assets/images/<clientName>-logo.png`
4. עדכן את `src/theme/assets.ts` עם הלוגו החדש
5. הרץ: `npm run start:<clientName>`

### 🎨 מערכת לוגואים דינמית
האפליקציה משתמשת במערכת לוגואים דינמית שמחליפה את הלוגו אוטומטית:

- **מסך בית הלקוח** - מציג את הלוגו של הלקוח הפעיל
- **מסך בית המנהל** - מציג את הלוגו של הלקוח הפעיל
- **זיהוי אוטומטי** - המערכת מזהה את הלקוח לפי משתנה הסביבה `CLIENT`
- **ברירת מחדל** - אם לא נמצא לקוח, משתמש ב-`clientA`
- **מיפוי קל** - כל לקוח מקושר ללוגו שלו ב-`src/theme/assets.ts`
